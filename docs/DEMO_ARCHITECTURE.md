## Key Design Decisions

### 1. Why Rich Library for Dashboard?
- Native terminal rendering (no browser needed)
- Real-time updates with Live context
- Beautiful tables, colors, progress bars
- Works over SSH for remote demos
- Lightweight and fast

### 2. Why Batch Inserts?
- Reduce PostgreSQL transaction overhead
- Prevent overwhelming Debezium
- Control event generation rate
- More realistic production pattern

### 3. Why Multiple Load Tests?
- Show scalability progression
- Identify performance characteristics at different loads
- Demonstrate system behavior under stress
- Provide comparative metrics

### 4. Why Exact Timestamp Tracking?
- Prove sub-50ms latency claims
- Identify bottlenecks in pipeline
- Measure CDC performance accurately
- Provide audit trail

### 5. Why Event Generator vs Manual Inserts?
- Realistic data with Faker library
- Referential integrity (user_ids, activity_ids)
- Repeatable test scenarios
- Automated execution
